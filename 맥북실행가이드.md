# 🍎 맥북에서 챗봇 실행하기

## 📋 전체 절차 (맥북 기준)

### 1단계: 터미널 열기

**방법 1: Spotlight 검색 사용 (가장 빠름)**
1. 키보드에서 `Command(⌘) + Space` 동시에 누르기
2. "터미널" 또는 "Terminal" 입력
3. Enter 키 누르기

**방법 2: Finder 사용**
1. Finder 열기
2. 왼쪽 사이드바에서 "응용 프로그램" 클릭
3. "유틸리티" 폴더 열기
4. "터미널" 더블클릭

**방법 3: Launchpad 사용**
1. F4 키 누르거나 트랙패드에서 4손가락 모으기
2. "기타" 폴더 열기
3. "터미널" 클릭

---

### 2단계: Node.js 설치 확인

터미널이 열리면 다음 명령어를 입력하고 Enter:

```bash
node --version
```

**결과 확인:**

✅ **버전이 표시되면** (예: `v18.17.0` 또는 `v20.10.0`)
→ Node.js가 설치되어 있습니다! **3단계로 이동**

❌ **"command not found" 또는 "zsh: command not found: node" 오류가 나오면**
→ Node.js 설치가 필요합니다. 아래 **Node.js 설치 방법**을 따라하세요.

---

### 3단계: Node.js 설치 (필요한 경우만)

#### 방법 1: 공식 웹사이트에서 설치 (추천)

1. **Safari 또는 Chrome 브라우저 열기**
2. 주소창에 입력: `https://nodejs.org/`
3. **"LTS" 버전** 다운로드 버튼 클릭 (초록색 버튼)
   - LTS = Long Term Support (안정적인 버전)
4. 다운로드된 파일 확인:
   - 다운로드 폴더에 `node-vXX.X.X.pkg` 파일이 생성됨
5. **다운로드된 .pkg 파일 더블클릭**
6. 설치 마법사가 열리면:
   - "계속" 버튼 클릭
   - "설치" 버튼 클릭
   - 관리자 비밀번호 입력 (Mac 로그인 비밀번호)
   - 설치 완료까지 대기
7. **터미널을 완전히 종료하고 다시 열기** (중요!)
8. 다시 `node --version` 입력하여 확인

#### 방법 2: Homebrew로 설치 (고급 사용자)

터미널에서:
```bash
brew install node
```

Homebrew가 설치되어 있지 않다면:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

---

### 4단계: 프로젝트 폴더로 이동

터미널에 다음 명령어를 입력하고 Enter:

```bash
cd ~/Desktop/챗봇
```

또는 전체 경로로:
```bash
cd /Users/parkkun/Desktop/챗봇
```

**확인 방법:** 터미널 프롬프트에 경로가 표시됩니다:
```
parkkun@MacBook 챗봇 %
```

---

### 5단계: 의존성 패키지 설치 (처음 한 번만)

터미널에 다음 명령어를 입력하고 Enter:

```bash
npm install
```

**진행 과정:**
- 몇 분 정도 걸릴 수 있습니다
- 터미널에 많은 메시지가 스크롤됩니다 (정상입니다)
- 완료되면 마지막에 다음과 같은 메시지가 표시됩니다:
  ```
  added 500 packages, and audited 501 packages in 2m
  ```

**오류가 발생하면:**
- 인터넷 연결 확인
- 다음 명령어로 캐시 정리 후 다시 시도:
  ```bash
  npm cache clean --force
  npm install
  ```

---

### 6단계: 환경 변수 확인

터미널에 다음 명령어를 입력하고 Enter:

```bash
cat .env.local
```

**결과 확인:**

✅ **다음과 같이 표시되면 OK:**
```
# Google Gemini API Key
NEXT_PUBLIC_GEMINI_API_KEY=AIzaSyDgO2AxMYt-Fd7aMBA9Lu4o5ku0bGulNu4
```

❌ **"No such file or directory" 오류가 나오면:**
다음 명령어로 파일 생성:
```bash
echo 'NEXT_PUBLIC_GEMINI_API_KEY=AIzaSyDgO2AxMYt-Fd7aMBA9Lu4o5ku0bGulNu4' > .env.local
```

---

### 7단계: 개발 서버 실행 ⭐ 가장 중요!

터미널에 다음 명령어를 입력하고 Enter:

```bash
npm run dev
```

**성공하면 다음과 같은 메시지가 표시됩니다:**

```
   ▲ Next.js 14.2.5
   - Local:        http://localhost:3000
   - Ready in 2.3s

   ○ Compiling / ...
   ✓ Compiled / in 1.2s
```

**⚠️ 매우 중요:**
- **이 터미널 창은 절대 닫지 마세요!**
- 서버가 실행 중일 때만 브라우저에서 접속할 수 있습니다
- 서버를 중지하려면: `Control + C` (터미널에서)

---

### 8단계: 브라우저에서 확인

1. **Safari 또는 Chrome 브라우저 열기**
2. 주소창에 입력: `http://localhost:3000`
3. Enter 키 누르기 또는 "이동" 버튼 클릭

**성공하면:**
- "📰 뉴스 챗봇" 제목이 보입니다
- 검색창이 표시됩니다
- 이제 정상 작동합니다! 🎉

## 🔧 문제 해결 (맥북 특화)

### 문제 1: "포트 3000이 이미 사용 중입니다"

**해결 방법:**
1. 터미널에서 `Control + C`로 서버 중지
2. 다른 포트로 실행:
   ```bash
   PORT=3001 npm run dev
   ```
3. 브라우저에서 `http://localhost:3001` 접속

### 문제 2: "권한이 거부되었습니다" (Permission denied)

**해결 방법:**
스크립트 실행 권한 부여:
```bash
chmod +x 시작하기.sh
```

### 문제 3: 터미널에서 한글이 깨져 보입니다

**해결 방법:**
터미널 설정 변경:
1. 터미널 > 환경설정 (Preferences)
2. 프로파일 > 텍스트
3. 인코딩: UTF-8 선택

### 문제 4: "zsh: command not found: npm"

**해결 방법:**
Node.js가 제대로 설치되지 않았습니다. 3단계를 다시 확인하세요.
