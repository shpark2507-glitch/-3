## 방법 1: 자동 스크립트 사용 (추천)

터미널에서 다음 명령어 실행:

```bash
cd ~/Desktop/챗봇
./배포업데이트.sh
```

---

## 방법 2: 수동으로 업데이트

터미널에서 다음 명령어를 순서대로 실행:

### 1단계: 프로젝트 폴더로 이동
```bash
cd ~/Desktop/챗봇
```

### 2단계: Git 저장소 초기화 (필요한 경우)
```bash
git init
```

### 3단계: 원격 저장소 연결
```bash
git remote add origin https://github.com/shpark2507-glitch/-3.git
```

또는 이미 연결되어 있다면:
```bash
git remote set-url origin https://github.com/shpark2507-glitch/-3.git
```

### 4단계: 파일 추가
```bash
git add .
```

### 5단계: 변경사항 확인
```bash
git status
```

**확인 사항:**
- `.env.local` 파일이 목록에 **없어야** 합니다
- 수정된 파일들이 목록에 있어야 합니다

### 6단계: 커밋
```bash
git commit -m "Update: 모바일 반응형 디자인 적용"
```

### 7단계: 브랜치 설정
```bash
git branch -M main
```

### 8단계: GitHub에 푸시
```bash
git push -u origin main
```

## Vercel 자동 배포

GitHub에 푸시하면 Vercel이 자동으로:
1. 변경사항 감지
2. 새로 빌드
3. 배포 완료

일반적으로 1-2분 내에 완료됩니다.

---

## 배포 확인

### 1. GitHub 확인
- https://github.com/shpark2507-glitch/-3 에서 최신 커밋 확인

### 2. Vercel 확인
- Vercel 대시보드에서 배포 상태 확인
- 배포 URL: https://3-five-xi.vercel.app

### 3. 모바일 테스트
- 실제 모바일 기기에서 접속
- 또는 브라우저 개발자 도구에서 모바일 뷰로 테스트

## 문제 해결

### "Permission denied" 오류
```bash
# 권한 부여
chmod +x 배포업데이트.sh
```

### "Authentication failed" 오류
GitHub 인증이 필요합니다:
```bash
# GitHub CLI 사용
gh auth login

# 또는 SSH 키 설정
```

### "Repository not found" 오류
저장소 URL이 올바른지 확인:
```bash
git remote -v
```
