# 🔴 ERR_CONNECTION_REFUSED 오류 해결 방법

## 문제 원인
브라우저에서 "ERR_CONNECTION_REFUSED" 오류가 나타나는 이유는 **개발 서버가 실행되지 않았기 때문**입니다.

## ✅ 해결 방법

### 1단계: 터미널 열기
- macOS: `Cmd + Space` → "터미널" 입력 → Enter
- 또는 Finder에서 응용 프로그램 > 유틸리티 > 터미널

### 2단계: 프로젝트 폴더로 이동
터미널에 다음 명령어를 입력하고 Enter:

```bash
cd /Users/parkkun/Desktop/챗봇
```

### 3단계: Node.js 설치 확인
```bash
node --version
```

**결과:**
- ✅ 버전이 표시되면 (예: v18.x.x) → 4단계로 이동
- ❌ "command not found" → Node.js 설치 필요
  - https://nodejs.org/ 접속 → LTS 버전 다운로드 → 설치
  - 설치 후 터미널을 다시 열고 다시 시도

### 4단계: 의존성 설치 (처음 한 번만)
```bash
npm install
```

이 과정은 몇 분 걸릴 수 있습니다. 완료되면 "added XXX packages" 메시지가 표시됩니다.

### 5단계: 개발 서버 실행 ⭐ 중요!
```bash
npm run dev
```

**성공하면 다음과 같은 메시지가 표시됩니다:**
```
  ▲ Next.js 14.x.x
  - Local:        http://localhost:3000
  - Ready in X.Xs

  ○ Compiling / ...
  ✓ Compiled / in X.Xs
```

**⚠️ 중요:** 
- 이 터미널 창은 **열어둔 채로** 유지해야 합니다
- 서버를 중지하려면 `Ctrl + C`를 누르세요
- 서버가 실행 중일 때만 브라우저에서 접속할 수 있습니다

### 6단계: 브라우저에서 다시 접속
1. 브라우저에서 `http://localhost:3000` 새로고침 (F5 또는 Cmd+R)
2. 이제 정상적으로 페이지가 표시되어야 합니다!

## 🚀 더 빠른 방법: 자동 스크립트 사용

터미널에서 한 번에 실행:

```bash
cd /Users/parkkun/Desktop/챗봇
./시작하기.sh
```

이 스크립트가 모든 확인과 설치를 자동으로 수행합니다.

## ❓ 여전히 안 되나요?

### 체크리스트:
- [ ] 터미널에서 `npm run dev` 명령어를 실행했나요?
- [ ] 터미널에 "Ready" 또는 "Compiled" 메시지가 표시되나요?
- [ ] 터미널 창을 닫지 않고 열어두었나요?
- [ ] 브라우저 주소창에 정확히 `http://localhost:3000`을 입력했나요?

### 다른 포트 사용 중일 수 있습니다
포트 3000이 이미 사용 중이면 다른 포트로 실행됩니다. 터미널 메시지를 확인하세요:
- 예: `- Local: http://localhost:3001` → 브라우저에서 `http://localhost:3001` 접속

### 오류 메시지가 있다면
터미널에 표시된 오류 메시지를 복사해서 알려주시면 더 정확한 해결책을 제시할 수 있습니다.
