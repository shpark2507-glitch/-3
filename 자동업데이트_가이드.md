# 🔄 GitHub 자동 업데이트 가이드

## 설정 방법

### 1단계: 초기 설정 (한 번만 실행)

터미널에서 다음 명령어 실행:

```bash
cd ~/Desktop/챗봇
./자동업로드_설정.sh
```

또는 npm 스크립트 사용:

```bash
npm run setup
```

이 명령어가 다음을 수행합니다:
- ✅ Git 저장소 초기화
- ✅ GitHub 원격 저장소 연결
- ✅ 브랜치 설정
- ✅ API 키 파일 보호 확인

## 사용 방법

### 방법 1: 자동 업로드 스크립트 (추천)

코드를 수정한 후 터미널에서:

```bash
cd ~/Desktop/챗봇
```

또는 npm 스크립트 사용:

```bash

```

**커스텀 커밋 메시지 사용:**
```bash
./자동업로드.sh "모바일 UI 개선"
```

### 방법 2: Git 명령어 직접 사용

```bash
cd ~/Desktop/챗봇
git add .
git commit -m "Update: 변경사항 설명"
git push origin main
```

## 자동 업데이트 흐름

```
코드 수정
    ↓
./자동업로드.sh 실행
    ↓
변경사항 감지
    ↓
Git에 커밋
    ↓
GitHub에 푸시
    ↓
Vercel 자동 배포 시작
    ↓
배포 완료 (1-2분)
```

## 문제 해결

### "Git 저장소가 초기화되지 않았습니다"
```bash
./자동업로드_설정.sh
```

### "원격 저장소가 설정되지 않았습니다"
```bash
git remote add origin https://github.com/shpark2507-glitch/-3.git
```

### "Authentication failed"
GitHub 인증이 필요합니다:
```bash
# GitHub CLI 사용
gh auth login

# 또는 SSH 키 설정
```

### "Permission denied"
```bash
chmod +x 자동업로드.sh 자동업로드_설정.sh
```

## Vercel 자동 배포

GitHub에 푸시하면 Vercel이 자동으로:
1. 변경사항 감지
2. 새로 빌드
3. 배포 완료

배포 확인: https://3-five-xi.vercel.app

---

## 주의사항

1. **API 키 보호**
   - `.env.local` 파일은 절대 Git에 포함되지 않습니다
   - Vercel 환경 변수에 API 키를 설정해야 합니다

2. **커밋 전 확인**
   - `git status`로 변경사항 확인
   - `.env.local`이 목록에 없어야 합니다

3. **배포 확인**
   - GitHub에 푸시 후 Vercel 대시보드 확인
   - 배포 완료까지 1-2분 소요
